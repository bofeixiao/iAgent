GitHub Copilot: 在 VS Code 中，GitHub Copilot 提供了三种不同的交互模式：

## Ask、Plan、Agent 的区别

### Ask（问答模式）
- **只读型**，不会修改任何文件
- 适合提问、理解代码、获取解释
- 可以搜索和阅读工作区代码来回答问题
- 例如："这个函数是做什么的？"、"这段代码有什么 bug？"

### Plan（规划模式）
- **规划型**，生成分步实施计划但不直接执行
- 会分析代码库，制定详细的修改方案
- 用户可以审查、调整计划后再决定是否执行
- 适合复杂重构或大型功能开发前的规划阶段

### Agent（代理模式）
- **执行型**，可以直接读写文件、运行终端命令
- 自主探索代码库、编辑文件、执行命令、迭代修复错误
- 会持续工作直到任务完成
- 适合实际实现功能、修 bug、重构代码等需要动手的任务

## 简单对比

| 特性 | Ask | Plan | Agent |
|------|-----|------|-------|
| 读取代码 | ✅ | ✅ | ✅ |
| 修改文件 | ❌ | ❌（仅建议） | ✅ |
| 运行命令 | ❌ | ❌ | ✅ |
| 自主决策 | 低 | 中 | 高 |
| 典型用途 | 提问/学习 | 制定方案 | 实施任务 |

**总结**：Ask 用来问，Plan 用来想，Agent 用来做。
