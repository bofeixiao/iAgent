# iData AI创作平台 - 系统需求说明书

## 1. 项目概述

### 1.1 项目背景
基于青山AI创作平台的功能分析,复现一个面向C端用户的AI内容创作平台,支持多种AI应用场景。

### 1.2 项目目标
- 提供AI辅助内容创作能力
- 支持用户思维分析和风格训练
- 提供丰富的创作应用模板
- 会员体系和积分系统
- 支持1000并发用户

### 1.3 核心用户群体
- 自媒体创作者
- 保险行业从业人员
- 企业营销人员
- 个人内容创作者

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 用户系统
**功能描述**: 用户注册、登录、个人信息管理

**详细需求**:
- 用户注册(手机号/邮箱)
- 用户登录(支持手机号、邮箱、第三方登录)
- 个人信息管理(头像、昵称、密码修改)
- 用户VIP会员状态标识
- 积分余额显示

**数据字段**:
- 用户ID、用户名、手机号、邮箱
- 头像URL、注册时间、最后登录时间
- VIP等级、积分余额、会员到期时间

#### 2.1.2 应用中心(Applications)
**功能描述**: AI应用模板库,用户可以基于模板创建应用实例

**详细需求**:
- 应用模板列表展示
- 应用分类筛选
- 应用详情查看
- 热门应用推荐
- 实时创作热点展示
- 应用创建引导流程

**典型应用模板**:
1. **奇幻照相馆** - 图片风格转换
   - 选择图片风格(哈利波特、加勒比海盗、甄嬛传等15种风格)
   - 上传形象照片
   - 选择图片宽高比(3:4、9:16、1:1等)
   - 自定义特殊要求

2. **文案生成器** - 智能文案创作
3. **思维导图** - AI辅助思维整理
4. **营销文案** - 营销推广内容生成

**数据字段**:
- 应用ID、应用名称、应用描述、应用图标
- 应用分类、创建时间、使用次数
- 所需积分、应用配置JSON

#### 2.1.3 作品管理(Works/Ins tances)
**功能描述**: 用户基于应用创建的作品实例管理

**详细需求**:
- 作品列表展示(卡片式/列表式)
- 作品详情查看
- 作品编辑/删除
- 作品搜索和筛选
- 作品创作历史日历视图
- 创作统计(7天创作次数、排名百分比)

**数据字段**:
- 作品ID、作品标题、应用ID
- 输入参数JSON、输出结果
- 创建时间、更新时间、状态(处理中/完成/失败)
- 消耗积分数

#### 2.1.4 思维系统(Thinking)
**功能描述**: 用户创建思维问卷,AI分析写作风格

**详细需求**:
- 思维问卷填写(行业、自媒体情况等)
- 思维状态管理(未创建/已创建/已分析)
- 写作风格分析结果展示
- 思维详情查看
- 资源关联管理
- 思维配额管理(30天内5次)

**数据字段**:
- 思维ID、用户ID、行业信息
- 自媒体情况、问卷回答JSON
- 写作风格分析结果、创建时间
- 分析完成时间、状态

#### 2.1.5 精华内容(Articles)
**功能描述**: 优质创作内容聚合展示

**详细需求**:
- 精华内容列表
- 内容分类标签
- 内容推荐算法
- 内容详情页
- 用户评分和反馈

**数据字段**:
- 文章ID、标题、内容、作者ID
- 分类标签、发布时间、浏览数
- 点赞数、评分

#### 2.1.6 会员与积分系统
**功能描述**: VIP会员购买和积分管理

**详细需求**:
- VIP会员等级(普通/银卡/金卡/铂金卡)
- 会员购买和续费
- 积分充值和消费
- 积分明细查询
- 会员权益展示
- 积分批次管理

**VIP等级**:
- 普通用户: 5积分
- VIP会员: 不同等级不同积分配额

**数据字段**:
- 会员等级ID、等级名称、价格
- 有效期、权益描述
- 积分记录ID、用户ID、变动积分
- 变动类型(充值/消费/赠送)、关联订单ID

#### 2.1.7 营销活动系统
**功能描述**: 活动运营和推广

**详细需求**:
- 活动横幅展示
- 新春活动/推荐有奖活动
- 邀请奖励机制
- 优惠券系统
- 活动参与记录

**数据字段**:
- 活动ID、活动名称、活动类型
- 开始时间、结束时间、活动规则JSON
- 奖励配置、活动状态

### 2.2 辅助功能

#### 2.2.1 热点趋势
- 实时热搜话题展示
- 热点来源标识(抖音、微信、头条等)
- 热度值显示
- 热点时间戳

#### 2.2.2 Banner管理
- 活动横幅展示
- Banner点击跳转
- Banner轮播

#### 2.2.3 问卷调查
- 用户初次登录问卷
- 行业信息收集
- 自媒体情况了解

## 3. 非功能需求

### 3.1 性能需求
- **并发支持**: 系统需支持1000并发用户
- **响应时间**: 页面加载时间 < 2秒
- **AI处理时间**: 简单应用 < 10秒,复杂应用 < 30秒
- **数据库查询**: 单表查询 < 100ms

### 3.2 可用性需求
- **系统可用性**: ≥ 99.5%
- **故障恢复时间**: < 30分钟
- **数据备份**: 每日自动备份

### 3.3 安全性需求
- 用户密码加密存储(BCrypt)
- API接口鉴权(JWT Token)
- 敏感数据传输加密(HTTPS)
- SQL注入防护
- XSS攻击防护
- CSRF防护

### 3.4 扩展性需求
- 支持水平扩展(应用服务器、数据库)
- 微服务架构设计
- 缓存层设计(Redis)
- 消息队列支持(RabbitMQ/Kafka)

### 3.5 兼容性需求
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **移动端**: 响应式设计,支持iOS 13+, Android 8+
- **分辨率**: 1920x1080, 1366x768, 移动端375-430px宽度

## 4. 数据库设计要点

### 4.1 核心数据表
1. **用户表** (t_user)
2. **应用表** (t_application)
3. **作品实例表** (t_instance)
4. **思维表** (t_thinking)
5. **文章表** (t_article)
6. **会员等级表** (t_membership_level)
7. **积分记录表** (t_credit_log)
8. **订单表** (t_order)
9. **活动表** (t_campaign)
10. **热点表** (t_trending)

### 4.2 数据量估算
- 预估用户数: 10万
- 日均创作作品数: 5000
- 年度作品总量: 180万
- 积分记录: 200万/年

## 5. API接口设计要点

### 5.1 RESTful API规范
- GET: 查询资源
- POST: 创建资源
- PUT: 更新资源(全量)
- PATCH: 更新资源(部分)
- DELETE: 删除资源

### 5.2 核心API列表
```
/api/v1/auth/register         # 用户注册
/api/v1/auth/login            # 用户登录
/api/v1/users/{id}            # 用户信息
/api/v1/applications          # 应用列表
/api/v1/applications/{id}     # 应用详情
/api/v1/instances             # 作品列表
/api/v1/instances/create      # 创建作品
/api/v1/thinking              # 思维管理
/api/v1/articles              # 文章列表
/api/v1/membership/levels     # 会员等级
/api/v1/credits               # 积分管理
/api/v1/campaigns             # 活动列表
/api/v1/trending              # 热点趋势
```
